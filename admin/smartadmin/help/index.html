Start

  • Created : 03/05/2017
  • By : Design Buddy

Thanks for purchasing Nyasa Admin Template. If you have any question you can ask through comments. Also you can find a lot of useful information right there. Thanks so much!

SmartAdmin is a fully featured premium admin template built on top of awesome Bootstrap 3.3.5, modern web technologies HTML5, CSS3 and jQuery. The theme includes beautiful dashboards including many ready to use hand crafted components. The components can be used very easy on any page. The theme is fully responsive and customizable. All the code including css, and javascript is very easy and helps developer to turn this theme into real web application.

Quick Tips :

  1. ID and Class name - You may remove all ID or Class names which contain demo-, they are only used for demonstration.
  2. Plugins - Feel free to remove the plugins you won't use (along with their style).
  3. jQuery selectors - The main selector has been cached and used in all of plugins. To improve performance and load time, you don't need to create a new variable to get the main selector, just call the variables.
    Variable Selectors
    window $('window')
    body $('body')
    document $('document')
    bodyHtml $('body, html')
    container $('#container')
    contentContainer $('#content-container')
    navbar $('#navbar')
    mainNav $('#mainnav-container')
    aside $('#aside-container')
    footer $('#footer')
    scrollTop $('#scroll-top')

  4. Useful functions

    List of useful functions might help you.

    Function name Returning value
    screenSize xs sm md lg
    isMobile true or false
    transition To test whether or not the browser supports transition properties ? // Return : true or false
    randomInt To generate random numbers in a selected range (min and max).

  5. User Names

    All user names used in the template are fictional and only used for demostration. They were randomly generated at uinames.com.

Plugins

All the plugins can be found in assets directory. Each plugin folder is containing files from plugins as well as initilization file for the plugin if required. Somep plugins don't require any initialization, they can direclt act on html markup itself, but some requires explicit intilization. So for them you'll be able to see the initialization js file in plugin directory itself.

Application Module

We have crafted all the JS based components very carefuly. The theme follows prototype based approch which gives performance and efficiency. We have one common app module which is present in each and every page. This is called "jasmineApp", please see file js/jquery.app.js file. Note that when you are adding new page, you should add this file becuase it contains some common initialization code as mentioned below. 

 
	//initializing tooltip
	jasmineApp.prototype.initTooltipPlugin = function() {
		$.fn.tooltip && $('[data-toggle="tooltip"]').tooltip()
	},

	//initializing popover
	jasmineApp.prototype.initPopoverPlugin = function() {
		$.fn.popover && $('[data-toggle="popover"]').popover()
	},

	//initializing nicescroll
	jasmineApp.prototype.initNiceScrollPlugin = function() {
		//You can change the color of scroll bar here
		$.fn.niceScroll &&  $(".nicescroll").niceScroll({ cursorcolor: '#9d9ea5', cursorborderradius: '0px'});
	},

	//initializing knob
	jasmineApp.prototype.initKnob = function() {
		if ($(".knob").length > 0) {
			$(".knob").knob();
		}
	}
